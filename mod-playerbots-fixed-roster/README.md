# mod-playerbots-fixed-roster

Small AzerothCore module for fixed WSG filler bots on the 19 PvP server.

Install by linking this directory into the core modules directory before configuring/building:

```bash
ln -s /root/services/19pvp/mod-playerbots-fixed-roster /root/services/19pvp/core/modules/mod-playerbots-fixed-roster
```

The module reads `PlayerbotsDatabase.playerbots_fixed_roster` and logs in enabled character GUIDs through
`sRandomPlayerbotMgr.AddPlayerBot(...)`. It does not patch `mod-playerbots` and does not require
`AiPlayerbot.RandomBotAutologin`.

Roster data is generated by `tasks/refresh_db.ts` from the `WSG_BOT` Google Sheet tab into
`core_scripts/generated-playerbots-fixed-roster.sql`. Each bot account key is inferred as `team-class`, for example
`alliance-priest` or `horde-mage`. Team is inferred from race; `TEAM` can be supplied only as a validation hint.

Bot gear is generated into `playerbots_fixed_roster_item`:

- normal player starter items from `ITEM` rows where `SOURCE` is `Starter` are included for each bot's class
- bot-only class starter items can be added in `ITEM` with `SOURCE` set to `BotStarter` or `WsgBotStarter`
- per-bot or class extras can be added in a `WSG_BOT_ITEM` tab with `ID`, optional `AMOUNT`, `SLOT` (`team-class`
  account key), numeric legacy `SLOT`, `TEAM`, and `CLASSES`

Configured items are equipped or stored when a fixed roster bot logs in or when `.wsgbots reload` runs. Existing item
counts are respected, so reloads/logins only add missing counts.

After applying roster SQL, reload without restarting:

```text
.wsgbots reload
```
